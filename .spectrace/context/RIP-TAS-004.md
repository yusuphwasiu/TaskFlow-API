## User Story

**As a** admin
**I want** to define user roles
**So that** I can control access and permissions within the TaskFlow API

## Description

Implement user roles for the TaskFlow API: Admin, Editor, and Viewer.

## Acceptance Criteria

- [ ] AC-1: Given a new user registration, when the role is set as Admin, Editor, or Viewer, then the user should have access according to the specified permissions.
- [ ] AC-2: Given a role assignment attempt with an invalid role, when processed, then the system should return a 400 error with the message 'Invalid role specified'.
- [ ] AC-3: Given an existing user with Viewer role, when upgraded to Editor, then new permissions should reflect immediately without re-login.
- [ ] AC-4: Given a database error during role assignment, when the operation fails, then the system should return a 500 error with a message indicating 'Role assignment failed due to server error'.

## Subtasks

- [ ] ST-1: Design user role schema in database
- [ ] ST-2: Implement role-based access control logic
- [ ] ST-3: Add validation for user role inputs
- [ ] ST-4: Write unit tests for role-based access
- [ ] ST-5: Document role permissions in API docs
